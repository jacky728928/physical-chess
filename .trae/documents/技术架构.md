# 引力弹弓棋 · 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────┐
│           UI Layer (HTML/CSS)       │
│  - Canvas 渲染棋盘                   │
│  - DOM 控制面板                      │
├─────────────────────────────────────┤
│         Game Logic (JS)              │
│  - 物理引擎（引力计算、轨迹推演）      │
│  - 游戏状态管理                      │
│  - 回合控制、胜负判定                │
├─────────────────────────────────────┤
│          Data Model                 │
│  - 棋子：{x, y, mass, owner}        │
│  - 向量：{vx, vy}                   │
└─────────────────────────────────────┘
```

## 2. 技术选型

| 类别 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 HTML5 + Canvas + JS | 单文件原型，无需构建 |
| 渲染 | Canvas 2D API | 足够轻量，帧率无要求 |
| 样式 | 内联 CSS + CSS Variables | 深空主题风格 |

## 3. 数据模型

### 3.1 棋子 (Piece)

```javascript
{
  id: number,        // 唯一标识
  x: number,         // 当前位置 x (1-9)
  y: number,         // 当前位置 y (1-9)
  mass: number,      // 质量: 5/3/1
  owner: 'A' | 'B',  // 所属玩家
  type: '★' | '●' | '·'  // 类型符号
}
```

### 3.2 恒星 (Star)

```javascript
{
  x: 5, y: 5,        // 固定中心
  mass: 20           // 极大质量
}
```

### 3.3 游戏状态 (GameState)

```javascript
{
  pieces: Piece[],   // 所有棋子
  star: Star,        // 中心恒星
  currentPlayer: 'A' | 'B',
  turn: number,      // 当前回合
  selectedPiece: Piece | null,
  phase: 'setup' | 'play' | 'end',
  winner: 'A' | 'B' | null
}
```

## 4. 核心算法

### 4.1 引力计算

```
F = G × M1 × M2 / r²

其中:
- G = 0.5
- r = √((x1-x2)² + (y1-y2)²)
- 合力 = Σ(所有其他物体的引力向量)
```

### 4.2 轨迹推演

```
for step in 0..200:
    1. 计算所有引力源的合力 (F_total)
    2. 加速度 a = F_total / mass
    3. 速度 v_new = v_old + a × Δt
    4. 位置 pos_new = pos_old + v_new × Δt

    终止条件:
    - pos 进入某格整数坐标 → 停在最近格
    - pos 出界 (x<1 || x>9 || y<1 || y>9) → 逃逸
    - step >= 200 → 停在最近整数格
```

### 4.3 吃子判定

```
轨迹终止格 (endX, endY) 恰好有敌方棋子 → 移除该敌方棋子
```

### 4.4 胜负判定

```
条件1: 对方棋子全部被吃 → 胜
条件2: 对方棋子全部逃逸 → 胜
条件3: 回合 > 20 → 残存棋子总质量大者胜
```

## 5. 交互流程

| 阶段 | 操作 | 结果 |
|------|------|------|
| 布子阶段 | 点击空格子放置己方棋子 | 验证位置合法性 |
| 布子阶段 | 点击"开始游戏" | 进入回合阶段 |
| 回合阶段 | 点击己方棋子 | 选中并显示可拖拽范围 |
| 回合阶段 | 拖拽并释放 | 设置初速度，开始轨迹推演 |
| 回合阶段 | 轨迹动画播放 | 显示预测路径 |
| 回合阶段 | 轨迹终止 | 吃子/停留/逃逸，切换回合 |

## 7. 性能考虑

- 轨迹推演：最多200步，Δt=0.1，约20ms内完成
- Canvas渲染：60fps目标，简单的形状绘制无压力
- 整体：单文件无依赖，浏览器直接打开即可运行
